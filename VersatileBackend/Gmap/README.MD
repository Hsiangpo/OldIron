# Gmap 模块 (VersatileBackend)

## 简介

`Gmap`（Google Map）模块是 `VersatileBackend` 中专用于基于地图数据进行**企业基础信息测绘与补齐**的通用组件。
该模块不仅用于获取公司的官网地址，由于地图业务的特性，它还是我们在多个国家（如韩国、泰国、日本等）获取**公司本土化标准名称（韩文/泰文名）、官方联系电话、精准坐标**的核心数据源。

## 核心业务逻辑与难点

通过各个单独国家（如泰国、韩国）的爬虫经验总结，Google Maps 的企业抓取绝不能简单地“只搜一个公司名”，该模块内部封装了以下几大核心策略：

### 1. 组合维度的精准搜索 (Geographic Parameters)

直接在 Google Maps 搜索 "ABC Limited" 极容易出现跨国、跨大洲的杂音数据。
因此，本模块在接受各个 Python 执行端传来的请求时，会自动基于企业实体拼接地域参数。实际的搜索 Query 通常是**“公司短名 + 城市 (City) + 州/省 (Region) + 国家 (Country)”**的组合，以此限制搜索的地理边界，大幅提高首条命中的准确率。

### 2. 本土语言名的提取 (Multi-language Local Names)

很多东亚、东南亚国家的工商注册是以本国语言（如韩文、泰文）为主，但我们在黄页或部分爬取源拿到的是英文拼写。
本模块内置了对相应国家的小语种字符识别规则。当搜索匹配成功时，除了返回英文名外，还会专门提取出韩文或泰文的公司全称（如 `company_name_th` 或 `company_name_kr`），这对于后续到该国本土政府工商库去交叉验证极其重要。

### 3. 多重维度匹配与打分机制 (Scoring System)

由于地图返回的数据可能存在“重名店铺”或“同名异业”的情况，本模块内部使用了一套严格的候选池打分逻辑：

- 提取 Google 返回的长串内部 JSON (XSSI 结构、tbm=map 格式) 里的所有候选坐标点。
- 将每个坐标点的名称、官网域名与目标 Query 进行对比打分（剔除掉通用词（如 tech, group, asia 等）带来的误判），最后得出可信度分值。当达到安全阈值（如 Score > 45）时，才会认为成功寻址并抛出结果。

### 4. 数据清洗 (Clean & Format)

- 去除无效占位符、社交网站主页（Facebook, X, LinkedIn）、地图信息聚合页。
- 电话号码剔除区号与冗余长字符并标准化转换（如统一替换成 `+66` 或 `+82` 格式）。

## 目录结构

- **`src/`**：这是整个 `Gmap` 组件的核心主代码存放地。所有涉及到 Go 环境下的网络连接保持、反爬虫（Rate Limiting 等待重试）、解析 Google Maps 内部的无头 protobuf/JSON 通信结构、打分与组合查询的实现逻辑都在这里维护。外部直接调用该 src 下暴露的接口即可。
