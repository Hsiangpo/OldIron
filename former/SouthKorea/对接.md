## 一、项目概述

### 核心需求

最终交付的是：**公司名 + 代表者（CEO） + 邮箱**，CSV 格式，按日增量交付。

### 数据获取链路

```
数据源网站 → 爬取公司名 + CEO + 官网URL
                                    ↓
                          官网域名 → Snov.io API 查该域名下的邮箱
                                    ↓
                          最终产出：公司名 + CEO + 邮箱
```

- **公司名、CEO**：从各数据源网站直接爬取
- **官网**：从各数据源的详情页提取（不是每家公司都有官网）
- **邮箱**：通过 Snov.io 的 Domain Search API，用官网域名查出该公司的邮箱列表。**没有官网的公司查不到邮箱，不会出现在最终交付里**

## 二、架构与运行

### 三阶段流水线（Pipeline）

```
Phase 1: 列表爬取 → company_ids.jsonl（公司名 + CEO）
    ↓ （流式，边产出边消费）
Phase 2: 详情页爬取 → companies.jsonl（补充官网URL）
    ↓ （流式，边产出边消费）
Phase 3: Snov 邮箱查询 → companies_with_emails.jsonl（用官网域名查邮箱）
```

- catch、incheon、dart、saramin 支持**流水线并行**模式（Phase 2/3 同时跑）
- khia 为**四阶段并行**：列表 → 详情补齐 → Google Maps 补官网 → Snov
- 每个 Phase 有 checkpoint，支持断点续跑
- **没有官网的公司**在 Phase 3 会被跳过（无法查邮箱）

6. API 使用环境选「모두」（全部）

9. 去邮箱点验证链接
10. 登录后在「마이페이지」→「인증키」获取 API Key

## 四、已修复的 Bug

### Bug 1: mois.go.kr 污染（严重，已修复）

**文件**: `src/korea_crawler/sites/incheon.py` → `_parse_detail_html()`

**问题**: incheon 详情页解析器的备选逻辑遍历所有外部链接，把页面 footer 的政府横幅 `mois.go.kr`（行政安全部）误当成公司官网。

**影响**: 24,831 条中有 19,448 条（78.3%）的官网被设成了政府网站，导致 Snov 查到的邮箱全是政府邮箱。

**修复**:

- 删除了"遍历所有外部链接"的备选逻辑
- 只从 `<p class="btn-box">` 内的"홈페이지"链接提取
- xpath 用 `contains(., ...)` 替代 `contains(text(), ...)`（因为文字在嵌套 `<span>` 内）

### Bug 2: incheon 串行等待（已修复）

**问题**: Phase 3（Snov）必须等 Phase 2（详情）全部跑完才开始。

**修复**: 改为流水线轮询模式，和 catch 一致。Phase 2 边写 companies.jsonl，Phase 3 每 10 秒轮询消费新数据。

### Bug 3: incheon TLS/连接抖动导致 Phase 2 中断（已修复）

**现象**: 详情阶段偶发 `curl (35)/(28)/(56)`，在旧逻辑下可能冒泡到 Phase 2 顶层导致整轮中断。

**修复**:

- `incheon_client.py` 将重试提升为 4 次，`curl (35)/(28)` 时重建 session 后重试
- 增加指数退避 + 抖动，补充 5xx 重试
- `incheon.py` 详情 worker 改为捕获所有异常并继续处理其他任务
- 详情并发默认从 8 调为 4，并增加 `--detail-concurrency` 参数

### Bug 4: 断点落后导致重复写入（已修复）

**现象**: 异常中断时，输出文件已写入但 checkpoint 尚未落盘，重启后重复追加同一 `comp_id`。

**修复**:

- 详情阶段启动时，将 `companies.jsonl` 里的 `comp_id` 合并进已处理集合
- Snov 阶段启动时，将 `companies_with_emails.jsonl` 里的 `comp_id` 合并进已处理集合
- 修复后新运行不会继续放大重复行

## 五、交付系统

### 去重规则

- **站内去重**: 各站点 Snov 完成后按域名去重（`dedup.py`）

## 九、注意事项

1. **incheon 网站不稳定**: 偶尔 TLS 错误 (curl 35)/超时(28)/断连(56)，客户端自动重试（最多 4 次）并在 TLS/超时时重建 session
2. **DART 官方条款**: 不允许一个人多号，但个人注册只需邮箱，技术上用不同邮箱可行
3. **checkpoint 文件**: 不要手动删除 `checkpoint_*.json`，否则会全量重跑
4. **Windows 路径**: 项目用 pathlib，跨平台兼容
5. **临时文件可删除**: 项目根目录下的 `check_mois.py`、`fix_incheon.py` 是调试脚本，可以删
6. **output/incheon/\*.bak**: 修复 mois.go.kr bug 前的备份文件，确认数据正常后可删
7. **Snov 额度**: 免费账号有月度限额，注意控制用量
8. **交付重跑**: `python product.py day1` 可以反复跑，会覆盖上一次的 day1 交付包
9. **0 条处理日志含义**: 例如“从第 2762 页开始, 上限 2761 页”+“Phase 2 总计处理 0 条”，表示断点已完成，不是异常
10. **saramin 官网字段**: 部分公司详情页没有 홈페이지，已接入 `render-homepage` 兜底，但仍会有无官网公司（这类公司无法走 Snov）
11. **khia 官网字段策略**: 站点详情页官网稀缺，现已接入 Google Maps 协议补官网；若短测用了 `--max-items`，`companies_with_emails.jsonl` 不会被覆盖，避免污染全量结果
